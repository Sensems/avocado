# 小程序私有化 CI/CD 构建平台 - 前端页面开发文档

本文档基于 PRD 提供的原型概念及现有后端 API 接口，梳理前端需要开发的页面结构、组件层级以及关联的接口绑定。

## 1. 路由与页面整体骨架 (Vue Router Structure)

系统采用典型的左右分栏（Side Menu + Header + Main Content）后台管理布局。

```text
/                      # 根路由 (Redirect 到 /dashboard)
├─ /login              # 登录页 (无 Layout)
├─ /* (拥有 Layout 的业务页面)
│  ├─ /dashboard       # 全局工作台 (大盘)
│  ├─ /projects        # 项目管理
│  │  ├─ /             # 项目列表页
│  │  └─ /:id          # 单个项目详情聚合页 (四 Tab)
│  ├─ /admin           # 管理员专属域 (需要 is_super_admin=true)
│  │  ├─ /users        # 用户管理
│  │  ├─ /credentials  # Git 凭证库
│  │  ├─ /robots       # IM 通知机器人
│  │  └─ /audit-logs   # 操作审计日志
│  └─ /404             # Not Found
```

## 2. 页面与组件开发明细

### 2.1 登录页 (`/views/login/index.vue`)

- **功能描述**：用于用户账密登录。
- **页面组件**：
  - `LoginForm`：包含用户名、密码输入框的表单组件。
- **关联 API**：
  - `POST /auth/login`: 传入 `{ username, password }`，获取并存储 JWT Token。
  - `GET /auth/profile`: 登录成功后紧接着获取当前登录用户信息及权限实体。

### 2.2 工作台大盘 (`/views/dashboard/index.vue`)

- **功能描述**：登录后的首屏，展示核心数据概览、排队构建任务及快捷入口。
- **页面组件**：
  - `QueueStatusCard`: 展示当前正在运行/等待的构建任务 (Robot ID 占用情况)。
  - `ProjectQuickList`: 以卡片形式展示“我参与的项目”及其最后一次构建状态。

### 2.3 系统设置域 (`/views/admin/*`)

此目录下所有页面必须在路由 Router Guard 和侧边栏菜单处做拦截，仅 `is_super_admin` 可见。

#### 用户管理 (`/views/admin/users/index.vue`)
- **功能描述**：管理团队成员账号。
- **页面组件**：
  - `UserTable`：展示用户列表 (`id`, `username`, `status`, `is_super_admin`)。
  - `CreateUserDialog`：新建用户的表单弹窗。
- **关联 API**：
  - `GET /users`: 获取用户分页列表。
  - `POST /users`: 创建新用户。
  - `PUT /users/:id/status`: 启用/禁用用户。

#### Git 凭证库 (`/views/admin/credentials/index.vue`)
- **功能描述**：统一录入与管理 SSH Keys 或 HTTPS Tokens。
- **页面组件**：
  - `CredentialTable`：列表隐去私钥明文，仅显示名称、创建者。
  - `CredentialFormDialog`：录入新凭证表单。
- **关联 API**：
  - `GET /git-credentials`: 列表。
  - `POST /git-credentials`: 新增。
  - `DELETE /git-credentials/:id`: 删除。
  - `POST /git-credentials/:id/test`: 触发连通性验证。

#### IM 通知机器人 (`/views/admin/robots/index.vue`)
- **功能描述**：录入企微、钉钉等 webhook 机器人。
- **页面组件** & **关联 API**：同凭证库类似 (`GET/POST/DELETE /im-robots`)，增加 `POST /im-robots/:id/test` 做测试发送。

### 2.4 项目管理域 (`/views/projects/*`)

#### 项目列表页 (`/views/projects/index.vue`)
- **功能描述**：展示用户有权限访问的小程序项目。
- **页面组件**：
  - `ProjectCardList`: 根据框架类型（原生/UniApp）展现不同标识，点击后跳转至详情页 `/projects/:id`。
  - `CreateProjectDialog`: 仅 Maintainer 或 Admin 显示新建按钮。
- **关联 API**：
  - `GET /projects`: 获取项目列表。
  - `POST /projects`: 创建新项目。

#### 项目详情聚合页 (`/views/projects/[id].vue`)
- **功能描述**：项目维度的控制中心。页面顶部展示项目概要（名称、Git URL、当前框架），主体采用 **ElTabs** 组件横向切换四个核心 Tab。
- **关联 API (页面初始化)**：
  - `GET /projects/:id`: 获取项目详细配置。

**Tab 1: 构建历史 (Build History)**
- **组件**: `BuildHistoryTable`
- **功能**: 展示版本号、提交信息、触发人、耗时、状态图标。提供“下载产物”和“查看二维码”按钮。
- **关联 API**:
  - `GET /build-tasks?projectId={id}`: 获取任务记录。
  - 点击构建记录触发对应的 `/artifacts` 产物下载/二维码获取 API。

**Tab 2: 实时日志终端 (Web Terminal)**
- **组件**: `TerminalLog` (基于 Xterm.js 封装)
- **功能**: 
  - 顶部栏：【触发构建】按钮、Branch/Commit Hash 输入选单。
  - 主体：连接 WebSocket 后实时渲染输出的流式日志。
- **关联 API**:
  - `POST /build-tasks`: 触发新建构建任务。
  - [WebSocket URL]: 连接后监听日志推送事件 `logData` 等。

**Tab 3: 成员管理 (Members)**
- **组件**: `MemberTable`, `AddMemberDialog`
- **功能**: 管理项目内角色的分配 (Maintainer / Developer / Guest)。
- **关联 API**:
  - `POST /projects/:id/members`: 拉新人进项目。
  - `PUT /projects/:id/members/:userId`: 修改角色。
  - `DELETE /projects/:id/members/:userId`: 移除出项目。

**Tab 4: 项目设置 (Settings)**
- **组件**: `ProjectConfigForm`
- **功能**: 包含基础信息设置、编译命令、产物路径、Git 凭证绑定下拉框、IM 机器人多选绑定的表单聚合页。只能在拥有 Maintainer 权限时点击保存。

## 3. 全局公共组件定义 (Public Components)

为了提效，以下 UI 交互必须抽离为全局组件：

1. **`RoleAuthBlock (组件级别权限守卫)`**
   - **Props**: `requireRole: 'admin' | 'maintainer' | 'developer'`
   - **功能**: 包裹在某块 UI 外层，当前用户角色若不满足，则渲染为空（`v-if`）。

2. **`WebSocketTerminal (黑客终端模拟器)`**
   - **Props**: `taskId: string | number` (当前构建任务 ID)
   - **功能**: 注入 taskId 后，自动握手建立 WS 连接，通过 xterm.js 渲染彩色输出。提供自动滚屏、断开重连逻辑。

3. **`FrameworkIcon`**
   - **Props**: `type: 'native' | 'uniapp'`
   - **功能**: 内置微信小程序 Logo 和 Vue/Uni-App Logo 的映射渲染，用于项目卡片。
