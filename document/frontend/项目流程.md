# 小程序私有化 CI/CD 构建平台 - 项目流程文档

本文档基于 PRD 梳理了平台中主要角色的操作流程，帮助前端团队整体理解各业务模块的串联逻辑。

## 1. 核心业务流程全景

按照系统的日常流转，整个平台的生命周期可以分为以下几个关键阶段：

1.  **基础设施准备 (系统初始化配置)**
2.  **项目创建与配置 (项目域初始化)**
3.  **日常构建与协作 (核心工作流)**
4.  **历史追溯与审计 (辅助工作流)**

---

## 2. 阶段化流程详述

### 2.1 基础设施准备 (Super Admin 视角)

在任何项目开始之前，系统管理员（Super Admin）需要搭建好系统运作的基础环境。

-   **步骤 1：用户入驻与管理**
    -   系统不支持公开注册。Admin 进入后台【用户管理】页面。
    -   创建开发团队成员账号，并分配默认初始密码。
    -   必要时对离职或调岗成员进行“账号禁用”操作。
-   **步骤 2：录入 Git 凭证库**
    -   Admin 进入【凭证管理】页面。
    -   输入拥有代码仓库拉取权限的 SSH Private Key 或 HTTPS Token。
    -   可以点击“连通性测试”确保凭证有效。该凭证将被池化，供后续项目引用，实现密钥不落开发者本地电脑。
-   **步骤 3：配置 IM 通知机器人**
    -   Admin 进入【机器人管理】页面。
    -   录入企微/钉钉/飞书等群聊机器人的 Webhook。
    -   发送测试消息，确保机器人能正常在群内播报。

### 2.2 项目创建与配置 (Maintainer 视角)

项目负责人（Maintainer）接手管理具体的业务项目，将其接入 CI/CD 平台。

-   **步骤 1：创建项目基本信息**
    -   进入【项目列表】并点击“创建项目”。
    -   填写项目名、AppID (小程序)、代码仓库地址 (Repo URL)。
-   **步骤 2：区分项目框架**
    -   选择框架类型：`Native`（原生小程序）或 `Uni-app`。
    -   如果是 Uni-app，必须填写编译命令（如 `npm run build:mp-weixin`）和编译产物路径（如 `dist/build/mp-weixin`）。原生项目则无此步骤。
-   **步骤 3：关联资源与依赖**
    -   在项目配置中，选择下拉菜单，绑定 2.1 阶段由 Admin 创建的【Git 凭证】。
    -   多选绑定需要接收此项目构建通知的【IM 机器人】。
-   **步骤 4：分配项目成员与角色**
    -   在【项目设置-成员管理】中，从系统用户库中拉人进项目。
    -   赋予 `Developer`（可构建）或 `Guest`（仅可看、可扫码）角色。

### 2.3 日常构建与协作 (Developer/Guest 视角)

这是平台最高频的使用场景。

-   **方式 A：手动触发构建**
    -   Developer 登录进入【工作台】或【项目详情】。
    -   点击“触发构建”按钮。
    -   选择需要检出的分支（Branch）或填写必要的构建备注。
-   **方式 B：代码提交自动触发 (Webhook)**
    -   Developer 在本地执行 `git push`（并在 commit 中带上类似 `[build]` 的关键词）。
    -   GitLab/GitHub 的 Webhook 触达平台后台，系统自动发起构建任务。
-   **构建过程追踪**
    -   Developer 进入项目详情里的【实时日志】Tab。
    -   页面呈现类似黑色 Terminal 的面板，通过 WebSocket 流式输出当前的构建步骤：`Git Pull` -> `Npm Install` -> `Compile` -> `Upload`。
-   **产物接收**
    -   构建成功后，之前绑定的 IM 机器人会在对应群聊中推送包含“体验版二维码”和更新说明的新消息。
    -   QA（测试人员）或 PM 可以直接在群里扫码体验，或者登录平台进入【构建历史】查看并扫码。

### 2.4 历史追溯与审计 (全员及 Admin 视角)

-   **历史版本管理**
    -   用户可以在【构建历史】中查阅往期版本，平台默认保留最近 10 次成功构建的源码包与二维码供下载、回滚查阅。
-   **安全审计**
    -   Super Admin 可以进入【审计日志】页面，查看全站的关键操作记录（如：谁在什么时间修改了某个项目的编译命令，或更新了关键凭证）。

## 3. 状态流转图示 (构建任务生命周期)

创建任务 (Pending) -> 排队等待 (Queued) -> 分配机器人执行 (Running)
-> [通过 WebSocket 推送日志]
-> 执行结果:
   - 成功 (Success) -> 保存产物 -> 触发 IM 通知
   - 失败 (Failed) -> 记录错误日志 -> 触发 IM 告警
   - 中断/取消 (Canceled)
