# 小程序私有化 CI/CD 构建平台 - 前端技术选型报告

根据 PRD 文档中描述的项目需求、非功能性需求（如高性能、实时 WebSocket 日志）以及“10年以上前端架构师”对代码质量和类型的严格要求，制定以下前端技术选型及架构规范。

## 1. 核心技术栈

| 技术域 | 选型 | 理由 |
| :--- | :--- | :--- |
| **基础框架** | **Vue 3** (Composition API) | 官方推荐范式，对 Hooks 的支持更轻量，响应式精准，适合后台管理系统的快速迭代，且在国内开发者生态极佳。 |
| **语言** | **TypeScript** (Strict Mode) | PRD 强调类型安全。全面使用 TS 接口定义 API Payload、响应体模式，杜绝 `any`。 |
| **构建工具** | **Vite** | 极速的冷启动和 HMR (热重载)，极大提升开发体验。符合现代 ES Module 驱动的工程化标准。 |
| **UI 组件库** | **Element Plus** | Vue 3 体系下最成熟的 B 端 UI 库，内置充足的常用组件（表格、表单、对话框），降低基础 UI 开发成本。 |
| **状态管理** | **Pinia** | Vue 官方推荐的状态管理库，完美支持 TS 推导，摒弃了 Vuex 繁琐的 Mutations，架构更简洁。 |
| **路由管理** | **Vue Router 4** | Vue 生态标准路由，支持嵌套路由、路由守卫（对于 RBAC 控制至关重要）。 |

## 2. 关键业务场景技术方案

### 2.1 实时构建日志 (Web Terminal)
PRD 要求：*“类似 Jenkins 的流式日志终端，WebSocket/SSE 推送”*。
- **通信方式**：使用原生的 `WebSocket` API 或 `socket.io-client` 建立与服务端的长连接，接收流式日志。
- **UI 渲染**：使用 **[Xterm.js](https://xtermjs.org/)**。这是最成熟的前端终端模拟器组件，性能极高，自带 ANSI 转义颜色的解析，可以完美呈现类似黑客帝国的原生命令行构建日志视觉体验，而非简单粗暴的 `<textarea>`。

### 2.2 权限控制与隔离 (RBAC)
PRD 要求：*“区分 Super Admin 与普通用户...区分项目的 Maintainer 和 Developer”*。
- **路由级拦截 (Vue Router Guards)**：
  - 维护动态路由表，根据用户登录后的全局角色 (`is_super_admin`) 决定是否加载 `/admin/*` 下的路由模块。
- **组件/按钮级可见性 (Custom Directive)**：
  - 开发自定义指令 `v-permission` 或专属组件 `<ActionAuth requiredRole="maintainer">`。
  - 在渲染核心按钮（如项目设置、修改成员）时进行校验，防止越权展示。

### 2.3 数据请求与一致性
- **请求封装**：基于 `Axios` 进行深度封装。
  - 全局拦截 401（Token 失效跳登录）和 403（越权访问提无权限）。
  - 自动在 Headers 中注入 Bearer Token。
  - 结合 Element Plus 的 `ElMessage` 统一提示后端抛出的常规错误信息。
- **防御性编程**：
  - 强烈推荐结合 TypeScript 的 Utility Types，如 `Partial<ProjectDto>` 等复用后端接口定义。
  - 对于深层对象访问，一律采用 Optional Chaining (`?.`) 防止运行时 TypeError。

## 3. 工程化与代码规范约束

作为一个对代码质量有极高要求的架构栈，我们将采用以下工具链闭环：

1. **规范校验 (Linting)**：
   - 引入 **ESLint** (严格规则) + **Prettier**。
   - 配置 `vue/setup-compiler-macros` 防止未声明引发的警告。
2. **提交钩子 (Git Hooks)**：
   - 使用 **Husky** + **lint-staged**。
   - 在 `git commit` 前，自动执行 ESLint 修复和 TypeScript 类型检查(`tsc --noEmit`)，从源头上阻断烂代码入库。
3. **命名规范**：
   - 组件：PascalCase (`ProjectCard.vue`)
   - 组合式函数：use 开头 (`useProjectList.ts`)
   - 类型/接口：PascalCase (`export interface IUser {}`)，或者大写开头代表类型。

## 4. 目录结构规划 (src)

```text
src/
├── api/             # 按业务模块划分的接口请求 (e.g., project.api.ts)
├── assets/          # 静态图片、全局 CSS 变量 (tailwind/CSS modules)
├── components/      # 全局复用组件 (e.g., TerminalLog.vue, AccessControl.vue)
├── composables/     # 抽取封装的 Vue Composable Hooks (e.g., useWebSocket)
├── layout/          # 大框架布局 (侧边栏、顶栏)
├── router/          # 路由配置及导航守卫
├── store/           # Pinia 状态管理片 (modules)
├── types/           # 全局或共用的 TypeScript 接口声明 (.d.ts)
├── utils/           # 纯函数工具库 (e.g., formatDate, aesEncrypt)
└── views/           # 顶层页面级组件
    ├── login/       # 登录页
    ├── dashboard/   # 大盘工作台
    ├── admin/       # Super Admin 专属配置页面
    └── projects/    # 项目详情、构建流水线 Tab 体系
```
